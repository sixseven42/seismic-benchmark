# 地震数据处理论文整理模板

本文档用于将你调研的论文统一整理成 SeismicBench 网站所需的 JSON 数据格式。

---

## 一、五大类别与系统任务值的映射

你的五个调研类别对应到网站系统中的 `tasks` 字段值如下：

| 你的分类 | 系统任务值 (task) | 说明 |
|---------|------------------|------|
| 插值/重构 | `interpolation` | 地震数据道插值、重构、数据恢复 |
| 随机噪声压制 | `denoising` | 使用 `tags` 区分："Random Noise" |
| 相干噪声压制 | `denoising` | 使用 `tags` 区分："Coherent Noise", "Ground Roll" |
| 初至拾取 | `first_arrival_picking` | 初至波自动拾取、震相识别 |
| 超分辨率 | `super_resolution` | 地震数据超分辨率重建、道加密 |

> **注意**：随机噪声压制和相干噪声压制在系统中都属于 `denoising` 任务，通过 `tags` 数组中的标签来区分具体子类型。

---

## 二、数据文件关系

网站使用 4 个 JSON 文件存储数据，它们之间的关系如下：

```
papers.json (论文)  ──introduces_model──►  models.json (方法)
     │                                          │
     │                                          │
     └────────────  results.json  ──────────────┘
                    (实验结果)
                      │
                      ▼
               benchmarks.json (数据集)
```

- **papers.json**：存储论文元数据。`introduces_model` 字段指向该方法在 `models.json` 中的 `id`。
- **models.json**：存储方法/模型元数据。`id` 被 `papers.json` 和 `results.json` 引用。
- **benchmarks.json**：存储数据集信息。`id` 被 `results.json` 引用。
- **results.json**：存储实验结果。连接 `model_id` 和 `benchmark_id`。

## 三、论文填写模板 (papers.json)

每篇论文按以下格式填写：

```json
{
  "id": "作者姓氏-方法名-年份",
  "title": "论文完整标题",
  "authors": "作者1, 作者2, 作者3",
  "org": "作者所在机构",
  "venue": "期刊或会议名称",
  "year": 2024,
  "abstract": "论文摘要（150-300字）",
  "tasks": ["任务值"],
  "tags": ["方法类型", "具体技术", "数据维度", "噪声类型（如适用）"],
  "arxiv_url": "https://arxiv.org/abs/xxxx.xxxxx",
  "doi": "https://doi.org/xx.xxxx/xxxx",
  "code_url": "https://github.com/xxxx/xxxx",
  "citation_count": 0,
  "introduces_model": "对应 models.json 的 id",
  "is_sota": false
}
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 唯一标识，建议格式：`作者名-方法缩写-年份`，如 `zhang-dncnn-2019` |
| `title` | 是 | 论文完整标题 |
| `authors` | 是 | 作者列表，用逗号分隔 |
| `org` | 是 | 主要机构，如 "Tongji University", "Stanford University" |
| `venue` | 是 | 期刊或会议，如 "Geophysics", "IEEE TGRS", "arXiv preprint" |
| `year` | 是 | 发表年份，数字 |
| `abstract` | 是 | 英文摘要，150-300字 |
| `tasks` | 是 | 数组，可选值：`interpolation`, `denoising`, `first_arrival_picking`, `super_resolution` |
| `tags` | 是 | 字符串数组，建议包含：方法类别、技术关键词、维度 |
| `arxiv_url` | 否 | arXiv 链接，没有则填 `null` |
| `doi` | 否 | DOI 链接，没有则填 `null` |
| `code_url` | 否 | 代码仓库链接，没有则填 `null` |
| `citation_count` | 是 | 论文引用次数，没有则填 `0` |
| `introduces_model` | 是 | 对应 `models.json` 中的 `id`，必须一致 |
| `is_sota` | 是 | 是否为当前最佳方法 (`true`/`false`) |

## 四、方法填写模板 (models.json)

每个方法按以下格式填写：

```json
{
  "id": "方法名-年份",
  "name": "方法展示名称",
  "authors": "作者简称",
  "org": "机构",
  "year": 2024,
  "emoji": "🎯",
  "type": "deep_learning",
  "tasks": ["任务值"],
  "description": "方法简介（50-100字）",
  "paper_url": "https://doi.org/xx.xxxx/xxxx",
  "code_url": "https://github.com/xxxx/xxxx",
  "weights_url": null,
  "is_open_source": true
}
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 唯一标识，与 `papers.json` 的 `introduces_model` 对应 |
| `name` | 是 | 展示名称，如 "DnCNN-Seis", "SeisU-Net" |
| `authors` | 是 | 简称，如 "Zhang & Lin", "Wang, Liu & Chen" |
| `org` | 是 | 机构 |
| `year` | 是 | 年份 |
| `emoji` | 是 | 展示用图标，如 🧹🔭🌊🎯🧠 |
| `type` | 是 | 方法类型：`traditional`（传统方法）, `deep_learning`（深度学习）, `hybrid`（混合方法） |
| `tasks` | 是 | 支持的任务数组，与 `papers.json` 一致 |
| `description` | 是 | 方法简介 |
| `paper_url` | 是 | 论文链接 |
| `code_url` | 否 | 代码链接，没有则填 `null` |
| `weights_url` | 否 | 预训练模型下载链接，没有则填 `null` |
| `is_open_source` | 是 | 是否开源 (`true`/`false`) |

## 六、实验结果填写模板 (results.json)

每条实验结果按以下格式填写：

```json
{
  "model_id": "方法在 models.json 中的 id",
  "benchmark_id": "数据集在 benchmarks.json 中的 id",
  "scores": {
    "snr": 28.4,
    "psnr": 41.2,
    "ssim": 0.94,
    "rmse": 0.021,
    "mse": 0.000441
  },
  "is_sota": false,
  "paper_url": "https://doi.org/xx.xxxx/xxxx",
  "code_url": "https://github.com/xxxx/xxxx",
  "date_added": "2024-03-15"
}
```

### scores 字段说明

不同任务需要填写的指标不同：

| 任务 | 必填指标 | 说明 |
|------|---------|------|
| `interpolation` | `snr`, `ssim`, `rmse` | SNR 越高越好，RMSE 越低越好 |
| `denoising` | `snr`, `ssim`, `rmse` | 同上 |
| `first_arrival_picking` | `accuracy`, `f1`, `mae` | Accuracy/F1 越高越好，MAE 越低越好 |
| `super_resolution` | `psnr`, `ssim`, `rmse` | PSNR/SSIM 越高越好，RMSE 越低越好 |

没有的数据填 `null` 或省略该字段。

## 八、整理 checklist

在导入每篇论文前，请确认：

- [ ] `papers.json` 中的 `id` 格式统一：`作者-方法-年份`
- [ ] `papers.json` 中的 `introduces_model` 与 `models.json` 中的 `id` 一致
- [ ] `tasks` 数组使用正确的系统值（`interpolation`, `denoising`, `first_arrival_picking`, `super_resolution`）
- [ ] 随机噪声/相干噪声论文的 `tasks` 都填 `denoising`，但通过 `tags` 区分
- [ ] `models.json` 中的 `type` 正确：`traditional` / `deep_learning` / `hybrid`
- [ ] `results.json` 中的 `model_id` 和 `benchmark_id` 都存在对应记录
- [ ] JSON 语法正确（可以使用 VS Code 或在线 JSON 校验工具检查）

## 十、系统已支持的类型标签建议

在填写 `tags` 时，建议使用以下已有标签或类似格式，便于后续筛选：

**方法类型**：`Traditional`, `Deep Learning`, `Hybrid`, `CNN`, `U-Net`, `Transformer`, `GAN`, `Diffusion`, `POCS`, `MSSA`, `Curvelet`, `F-K`, `Radon`

**数据维度**：`1D`, `2D`, `3D`

**数据类型**：`Synthetic`, `Field`, `Marine`, `Land`, `Public`

**噪声类型（去噪任务）**：`Random Noise`, `Coherent Noise`, `Ground Roll`, `Gaussian`

**采样类型（插值任务）**：`Random Missing`, `Regular Missing`
